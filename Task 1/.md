## What is NetSniff Pro?

NetSniff Pro is a command-line network packet analyzer that captures live traffic, decodes protocol layers from Ethernet all the way up to the application layer, and runs **real-time anomaly detection** to flag suspicious activity like port scans, DNS exfiltration attempts, and ICMP floods — all with zero pip installs.

It's designed to bridge the gap between `tcpdump` (powerful but raw) and Wireshark (GUI-heavy) — giving you a readable, color-coded, intelligent CLI sniffer that you can run instantly on any Linux machine.

## Features

### Core Packet Capture
- Raw socket capture using `AF_PACKET` — reads every frame off the wire
- Parses all major layers: **Ethernet → IP → TCP/UDP/ICMP → Application**
- Protocol detection by port: HTTP, TLS (with ClientHello fingerprinting), DNS, SSH, DHCP
- TCP flag parsing: SYN, ACK, PSH, FIN, RST, URG
- Supports any Linux network interface: `eth0`, `wlan0`, `lo`, `any`

### Anomaly Detection Engine (Unique)
NetSniff Pro watches packet streams in real time and raises alerts for:

| Anomaly | Detection Logic |
|---|---|
| **Port Scan** | Single source IP probing >15 unique destination ports |
| **DNS Exfiltration** | Subdomain labels >30 chars (classic data-over-DNS technique) |
| **High DNS Rate** | >20 DNS queries from one host within a 5-second window |
| **ICMP Flood** | >30 echo requests from one source within 2 seconds |
| **Suspicious Payload** | Large payloads (>8 KB) over non-standard ports |

### Rich CLI Output
- ANSI 256-color terminal output — each protocol gets a distinct color
- Per-packet line: `#  timestamp  [PROTO]  src:port → dst:port  bytes  flags  info`
- End-of-session statistics panel with ASCII bar charts for protocol distribution
- Top talkers ranked by bytes transferred
- Active conversation tracker (IP pairs + total data)

### Hex Dump Mode
When run with `--hex`, every packet is shown as a color-layered hex dump:
- 🟡 **Yellow** — Ethernet + IP header (first 34 bytes)
- 🔵 **Cyan** — TCP/UDP transport header
- 🟢 **Green** — Application payload

### JSON Export
Every captured packet can be exported to structured JSON, including: timestamp, src/dst IPs, ports, protocol, length, TTL, flags, and decoded info string. Useful for offline analysis, feeding into SIEM tools, or writing your own detection scripts.

## Installation

No installation required. NetSniff Pro uses **only Python's standard library** — `socket`, `struct`, `threading`, `json`, `argparse`, `signal`, `collections`.

```bash
# Clone the repo
git clone https://github.com/VP5412/netsniff-pro.git
cd netsniff-pro

# Confirm Python version (3.6+ required)
python3 --version

# Run — sudo is required for raw socket access
sudo python3 netsniff.py
```

> **Linux only.** Raw `AF_PACKET` sockets are a Linux kernel feature. Windows requires a different approach (Npcap + scapy).

## Usage

```
usage: netsniff.py [-h] [-i IFACE] [-c COUNT] [-f FILTER] [-v] [--hex]
                   [--json] [-o OUTPUT] [--anomaly-only]

options:
  -h, --help            show this help message and exit
  -i, --iface           Network interface to listen on (default: all)
  -c, --count           Number of packets to capture (0 = unlimited)
  -f, --filter          Simple protocol filter: tcp, udp, dns, http, etc.
  -v, --verbose         Show TTL, window size, payload length per packet
  --hex                 Show color-layered hex dump for every packet
  --json                Output packets in JSON format
  -o, --output          Save JSON to a file (auto-updates while capturing)
  --anomaly-only        Suppress normal output, only print anomaly alerts
```

### Examples

```bash
# Capture everything on all interfaces
sudo python3 netsniff.py

# Capture on WiFi adapter, verbose mode
sudo python3 netsniff.py -i wlan0 -v

# Capture exactly 100 packets then stop
sudo python3 netsniff.py -c 100

# Show hex dump — great for learning protocol structure
sudo python3 netsniff.py --hex -c 50

# Filter for DNS only
sudo python3 netsniff.py -f dns

# Save full session as JSON for offline analysis
sudo python3 netsniff.py --json -o capture.json

# Run anomaly detection silently — only print alerts
sudo python3 netsniff.py --anomaly-only

# Combine: WiFi, verbose, anomaly alerts, save JSON
sudo python3 netsniff.py -i wlan0 -v --anomaly-only --json -o session.json
```

## How It Works — Protocol Stack

NetSniff Pro manually parses each protocol layer using Python's `struct` module:

```
Raw Socket (AF_PACKET)
        │
        ▼
  Ethernet Frame          ← 14 bytes: src/dst MAC, EtherType
        │
        ├── 0x0800 ──▶  IPv4 Header          ← 20+ bytes: src/dst IP, TTL, proto
        │                     │
        │                     ├── Proto 6  ──▶  TCP Header    ← ports, seq, flags
        │                     │                     │
        │                     │                     ├── Port 80  ──▶  HTTP
        │                     │                     ├── Port 443 ──▶  TLS (fingerprinted)
        │                     │                     └── Port 22  ──▶  SSH
        │                     │
        │                     ├── Proto 17 ──▶  UDP Header    ← ports, length
        │                     │                     │
        │                     │                     └── Port 53  ──▶  DNS (name extracted)
        │                     │
        │                     └── Proto 1  ──▶  ICMP          ← type, code
        │
        └── 0x0806 ──▶  ARP                  ← sender/target IP
```

## Concepts Demonstrated

This project is a practical implementation of the following cybersecurity and networking concepts:

- **Raw socket programming** — bypassing the OS network stack to read frames directly
- **Binary protocol parsing** — using `struct.unpack` to decode fixed-format headers
- **OSI model in practice** — handling Layer 2 through Layer 7 in a single pipeline
- **Real-time stream processing** — multi-threaded capture with thread-safe stat updates
- **Anomaly detection algorithms** — sliding time windows, threshold-based alerting
- **TLS fingerprinting basics** — identifying ClientHello handshakes from raw bytes
- **DNS query analysis** — extracting QNAME from compressed DNS wire format
- **CLI UX design** — ANSI escape codes, columnar output, live stats panels
